# 智能座舱主交互系统 - 代码实现计划

## 技术栈确认

**平台**: Android 14 AAOS (Android Automotive OS)  
**架构**: MVVM + Repository Pattern  
**UI**: Jetpack Compose for Automotive  
**通信**: Car API (android.car.*)  
**依赖**: CarService 已按标准提供

### 阶段1: 核心框架搭建
1. 项目初始化与 Gradle 配置
2. Car API 连接管理器
3. 依赖注入框架 (Hilt)

### 阶段4: 集成与测试
1. 模块间通信

## AAOS 特定考虑

### Car API 使用
```kotlin
// 获取 Car 实例
val car = Car.createCar(context)
val carInfo = car.getCarManager(Car.INFO_SERVICE) as CarInfoManager
val carProperty = car.getCarManager(Car.PROPERTY_SERVICE) as CarPropertyManager
```

### 车辆属性监听
```kotlin
// 监听车速
val propertyIds = listOf(
    VehiclePropertyIds.PERF_VEHICLE_SPEED,
    VehiclePropertyIds.GEAR_SELECTION,
    VehiclePropertyIds.PARKING_BRAKE_ON
)
```

### 驾驶状态处理
- `CarDrivingStateEvent` - 驾驶状态变化
- `CarUxRestrictions` - UX限制管理
- 行驶中限制交互的逻辑

## 代码规范

1. **Kotlin 编码规范**: 遵循官方 Kotlin 编码规范
2. **架构**: 严格 MVVM，禁止在 View 中直接调用数据层
3. **Car API**: 所有车辆数据访问通过 VehiclePropertyHelper
4. **线程**: 车辆 I/O 使用专用线程池
5. **错误处理**: 车辆数据异常不能崩溃 UI
6. **注释**: 符合 ASPICE 文档追溯要求
