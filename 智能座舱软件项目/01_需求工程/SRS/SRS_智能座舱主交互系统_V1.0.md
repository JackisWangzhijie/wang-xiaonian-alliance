## 2. 系统概述

### 2.1 系统背景
智能座舱是汽车智能化发展的核心领域，主交互系统作为座舱的"大脑"，承担着信息展示、人机交互、应用管理、导航服务等关键功能。本项目旨在开发一套符合行业先进水平、满足车规级质量要求的智能座舱主交互软件系统。

### 2.2 系统目标
1. **用户体验目标**: 提供流畅、直观、安全的人机交互体验
2. **功能完整性目标**: 覆盖消息管理、导航、应用控制等核心功能
3. **性能目标**: 系统响应时间≤100ms，冷启动时间≤3s
4. **安全目标**: 符合ISO 26262 ASIL B等级要求
5. **网络安全目标**: 符合ISO/SAE 21434要求
6. **质量目标**: 软件缺陷密度≤0.1个/千行代码

## 3. 功能需求

### 3.1 需求标识与分类
每条需求采用以下标识格式：  
**REQ-[模块]-[类型]-[序号]-[ASIL等级]**

模块代码：
- MSG：消息中心
- DRV：行驶限制
- NAV：地图导航
- FWK：系统框架
- AND：Android系统
- MKT：应用市场
- AI：AI交互
- OTH：其他功能

类型代码：
- FUN：功能需求
- PER：性能需求
- SAF：安全需求
- CYB：网络安全需求
- INT：接口需求

### 3.2 消息中心模块

#### 3.2.1 消息管理功能
**REQ-MSG-FUN-001-QM**
- **需求描述**: 系统应提供消息统一管理功能，支持应用消息的分类、排序和筛选
- **输入**: 各应用消息源（导航、音乐、车辆等）
- **处理**: 消息分类、优先级计算、去重处理
- **输出**: 分类后的消息列表
- **验收准则**: 
  - 支持≥20个应用消息源接入
  - 消息分类准确率≥99%
  - 消息去重准确率100%

**REQ-MSG-FUN-002-QM**
- **需求描述**: 系统应支持消息的历史记录查询，保存时间≥30天
- **输入**: 用户查询条件（时间范围、应用类型、关键词）
- **处理**: 数据库查询、结果过滤
- **输出**: 历史消息列表
- **验收准则**: 
  - 30天内消息查询响应时间≤500ms
  - 历史消息完整性100%

#### 3.2.2 消息优先级管理
**REQ-MSG-FUN-003-ASIL_A**
- **需求描述**: 系统应实现消息优先级智能管理，驾驶相关消息优先级最高
- **优先级定义**:
  - P0（紧急）: 安全告警、碰撞预警
  - P1（高）: 导航关键提示、来电
  - P2（中）: 消息通知、应用更新
  - P3（低）: 资讯推送、活动提醒
- **验收准则**:
  - P0消息必须立即显示，延迟≤100ms
  - 驾驶模式下自动过滤P2/P3消息

#### 3.2.3 消息弹窗功能
**REQ-MSG-FUN-004-ASIL_A**
- **需求描述**: 系统应提供安全的消息弹窗机制，驾驶时不遮挡关键信息
- **安全要求**:
  - 弹窗位置避开驾驶视线区域
  - 驾驶时自动缩小弹窗尺寸
  - 提供语音播报替代弹窗显示
- **验收准则**:
  - 弹窗不遮挡前向视野区域
  - 驾驶模式弹窗面积≤屏幕10%

#### 3.2.4 活动资讯与通知管理
**REQ-MSG-FUN-005-QM**
- **需求描述**: 系统应支持活动资讯的展示和通知管理
- **功能要求**:
  - 支持图文混排展示
  - 支持富媒体内容（视频、GIF）
  - 用户可自定义通知偏好
- **验收准则**:
  - 资讯加载时间≤2s
  - 支持离线浏览已加载内容

#### 3.2.5 智能推荐管理
**REQ-MSG-FUN-006-QM**
- **需求描述**: 系统应提供基于用户行为的智能推荐功能
- **推荐场景**:
  - 常用应用快捷入口
  - 个性化内容推荐
  - 场景化服务推荐（加油、停车、充电）
- **验收准则**:
  - 推荐准确率≥80%
  - 推荐响应时间≤500ms

### 3.3 行驶限制模块

#### 3.3.1 中控行驶挡位应用控制
**REQ-DRV-FUN-007-ASIL_B**
- **需求描述**: 系统应根据车辆行驶状态自动限制中控应用操作
- **触发条件**:
  - 车速＞0km/h时启用限制
  - 挡位在D/R挡时启用限制
  - 驻车制动释放时启用限制
- **限制范围**:
  - 禁止视频播放（包括腾讯视频等流媒体）
  - 禁止应用市场下载/安装
  - 禁止游戏类应用启动
  - 限制浏览器滚动速度
- **验收准则**:
  - 限制触发延迟≤200ms
  - 限制生效准确率100%
  - 满足ISO 26262 ASIL B要求

**REQ-DRV-FUN-008-ASIL_B**
- **需求描述**: 系统应提供行驶限制的白名单管理功能
- **白名单内容**:
  - 导航类应用：高德地图、百度地图
  - 音乐类应用：QQ音乐、网易云音乐
  - 通讯类应用：蓝牙电话、语音助手
  - 车辆控制应用：空调、座椅调节
- **验收准则**:
  - 白名单应用功能不受限制
  - 白名单更新需OTA验证

#### 3.3.2 应用行为控制
**REQ-DRV-FUN-009-ASIL_A**
- **需求描述**: 系统应提供细粒度的应用行为控制功能
- **控制功能**:
  - 暂停：暂停视频/游戏运行，保留状态
  - 返回HOME：强制应用返回后台
  - 限制交互：仅保留语音交互
  - 灰度显示：降低非关键应用亮度
- **验收准则**:
  - 应用状态保存完整率100%
  - 控制指令响应时间≤100ms

#### 3.3.3 安全状态恢复
**REQ-DRV-FUN-010-ASIL_B**
- **需求描述**: 系统应在车辆停止后自动恢复应用正常状态
- **恢复条件**:
  - 车速=0km/h且持续＞3s
  - 挡位在P挡
  - 驻车制动启用
- **验收准则**:
  - 应用状态恢复延迟≤500ms
  - 状态恢复准确率100%

### 3.4 地图导航模块

#### 3.4.1 地图导航功能

#### 4.2.1 可用性

- **REQ-REL-002-ASIL_B**: 安全相关功能故障率≤10⁻⁵/小时

#### 4.2.2 容错性
- **REQ-REL-003-ASIL_B**: 单点故障不应导致系统完全失效

- **REQ-REL-006-ASIL_B**: 系统故障后恢复时间（MTTR）≤30分钟
- **REQ-REL-007-QM**: 应用崩溃后自动重启时间≤5s
