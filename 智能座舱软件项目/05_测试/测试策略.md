### 3.2 测试原则

#### AAA模式
```kotlin
@Test
fun `should show popup when driving stopped`() {
    // Arrange
    val message = createTestMessage(priority = P1_HIGH)
    every { mockDrivingState.isDriving() } returns false
    
    // Act
    controller.checkAndShow(message)
    
    // Assert
    verify { mockPopupManager.show(any(), any()) }
}
```

#### 命名规范
- 测试方法: `` fun `should [expected behavior] when [condition]`() ``
- 测试类: `{ClassName}Test`
- 测试文件: `{ClassName}Test.kt`

## 5. 系统测试策略

### 5.1 测试类型

| 类型 | 方法 | 工具 |
|------|------|------|
| 功能测试 | 手动+自动化 | Espresso/UI Automator |

| 兼容性测试 | 手动 | 多设备验证 |

### 5.2 Espresso测试示例

```kotlin
@Test
fun testMessagePopupDisplay() {
    // 发送测试消息
    onView(withId(R.id.send_test_message))
        .perform(click())
    
    // 验证弹窗显示
    onView(withText("测试消息标题"))
        .check(matches(isDisplayed()))
    
    // 验证关闭按钮
    onView(withId(R.id.btn_close))
        .perform(click())
    
    // 验证弹窗消失
    onView(withText("测试消息标题"))
        .check(doesNotExist())
}
```
