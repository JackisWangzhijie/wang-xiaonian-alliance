## 审查流程

```
┌──────────────────────────────────────────────────────────────────────┐
│  1. 提交前自检 ← 开发工程师                                          │
│     └─ 运行 preCommitCheck                                           │
├──────────────────────────────────────────────────────────────────────┤
│  2. 工具自动检查 ← CI/CD                                             │
│     └─ Detekt + Lint + Ktlint + 单元测试 + 安全扫描                  │
├──────────────────────────────────────────────────────────────────────┤
│  3. 同行评审 ← 至少1名开发工程师 (非作者)                            │
│     └─ 代码逻辑 + 设计 + 性能审查                                    │
├──────────────────────────────────────────────────────────────────────┤
│  4. 架构师审查 ← 关键模块/公共组件                                   │
│     └─ 架构一致性 + 扩展性 + 安全性                                  │
├──────────────────────────────────────────────────────────────────────┤
│  5. 安全工程师审查 ← 安全相关代码                                    │
│     └─ ASIL合规 + 威胁分析 + 安全机制                                │
├──────────────────────────────────────────────────────────────────────┤
│  6. 质量工程师审查 ← 合规性                                          │
│     └─ ASPICE + 追溯性 + 覆盖率                                      │
├──────────────────────────────────────────────────────────────────────┤
│  7. 批准合并 ← 维护者                                                │
└──────────────────────────────────────────────────────────────────────┘
```

```kotlin
suspend fun processSafetyCriticalOperation(): Result<Unit> {
    return try {
        // 执行安全关键操作
        performOperation()
        Result.success(Unit)
    } catch (e: SafetyException) {
        // 记录错误但不暴露敏感信息
        logSecurityEvent("Safety operation failed", e)
        // 进入故障安全状态
        enterFailSafeMode()
        Result.failure(UserFriendlyException("Operation failed"))
    } catch (e: Exception) {
        // 意外异常，记录详细信息供调试
        logInternalError("Unexpected error", e)
        enterFailSafeMode()
        Result.failure(UserFriendlyException("System error"))
    }
}
```

```kotlin
@Composable
fun MessageList(messages: List<Message>) {
    // 派生状态优化
    val sortedMessages by remember(messages) {
        derivedStateOf { messages.sortedByDescending { it.timestamp } }
    }
    
    LazyColumn {
        items(
            items = sortedMessages,
            key = { it.id }  // 指定key避免不必要的重组
```

## 附录

### 自动检查命令

```bash
# 完整质量门禁
./gradlew qualityGate

# 快速检查 (本地开发)
./gradlew preCommitCheck

# 仅静态分析
./gradlew detekt lintDebug

# 仅单元测试
./gradlew testDebugUnitTest

# 覆盖率检查
./gradlew jacocoTestCoverageVerification

# 安全扫描
./gradlew dependencyCheckAnalyze
```

### 严重度定义

| 严重度 | 定义 | 处理方式 |
|--------|------|----------|
| 🔴 阻塞 | 违反安全/合规要求，可能导致严重缺陷 | 必须修复，阻塞合并 |
| 🟠 高危 | 可能导致系统故障或安全风险 | 必须修复 |
| 🟡 中危 | 影响代码质量或维护性 | 建议修复，可技术债务 |
| 🟢 低危 | 轻微问题 | 可选修复 |

### 参考标准

- ASPICE v3.1
