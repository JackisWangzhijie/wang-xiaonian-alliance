## 审查流程

```
┌──────────────────────────────────────────────────────────────┐
│  1. 提交前自检 ← 开发工程师                                   │
│     └─ 运行 preCommitCheck                                   │
├──────────────────────────────────────────────────────────────┤
│  2. 工具自动检查 ← CI/CD                                      │
│     └─ Detekt + Lint + Ktlint + 单元测试                     │
├──────────────────────────────────────────────────────────────┤
│  3. 同行评审 ← 至少1名开发工程师                              │

├──────────────────────────────────────────────────────────────┤
│  6. 批准合并 ← 维护者                                         │
└──────────────────────────────────────────────────────────────┘
```

### D. 协程与并发

#### D.1 协程规范

| 检查项 | 检查点 | 通过标准 |
|--------|--------|----------|
| D1.1 | 避免GlobalScope | 使用结构化并发 |
| D1.2 | CoroutineScope正确 | 使用viewModelScope/lifecycleScope |
| D1.3 | Dispatchers指定 | IO/Default/Main明确指定 |
| D1.4 | 异常处理 | 使用SupervisorJob处理子协程异常 |

```kotlin
// ✅ 良好的协程使用
class MessageViewModel @Inject constructor(
    private val useCase: GetMessagesUseCase
) : ViewModel() {
    
    private val viewModelScope = CoroutineScope(
        SupervisorJob() + Dispatchers.Main + exceptionHandler
    )
    
    fun loadMessages() {
        viewModelScope.launch {
            useCase()
                .flowOn(Dispatchers.IO)
                .catch { error ->
                    _error.value = error.message
                }
                .collect { messages ->
                    _messages.value = messages
                }
        }
    }
}
```

@Composable
fun MessageList(messages: List<Message>) {
    val sortedMessages by remember(messages) {
        derivedStateOf { messages.sortedByDescending { it.timestamp } }
    }
    
    LazyColumn {
        items(
            items = sortedMessages,
            key = { it.id }
        ) { message ->
            MessageItem(message = message)
        }
    }
}

### 严重度定义

| 严重度 | 定义 | 处理方式 |
|--------|------|----------|
| 🔴 阻塞 | 违反安全/合规要求 | 必须修复 |
| 🟠 高危 | 可能导致严重缺陷 | 必须修复 |
| 🟡 中危 | 影响代码质量 | 建议修复 |
| 🟢 低危 | 轻微问题 | 可选修复 |
