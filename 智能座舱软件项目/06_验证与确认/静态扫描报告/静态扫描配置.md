# 智能座舱主交互系统 - 代码静态扫描配置

## 1. 配置概述

### 1.1 项目背景
- **项目名称**: 智能座舱主交互系统
- **开发语言**: Kotlin (Android)
- **质量等级**: ASIL-B (汽车功能安全)

### 1.2 静态扫描工具链

| 工具 | 用途 | 优先级 | 集成阶段 |
|------|------|--------|----------|
| SonarQube | 代码质量综合平台 | 高 | CI/CD持续集成 |
| Detekt | Kotlin静态分析 | 高 | 编译前预检查 |
| Android Lint | Android专用检查 | 高 | 编译时检查 |
| ktlint | 代码风格规范 | 中 | 提交前检查 |

## 2. 质量门禁 (Quality Gate)

### 2.1 阻塞级指标

| 指标类型 | 阈值 | 说明 |
|----------|------|------|
| 阻塞问题 (Blocker) | 0个 | 必须修复，否则禁止合并 |
| 严重问题 (Critical) | ≤5个 | 高优先级修复 |
| 主要问题 (Major) | ≤50个 | 计划内修复 |
| 代码重复率 | ≤3% | 新代码重复率 |
| 技术债务 | ≤5天 | 预计修复工时 |

### 2.2 汽车软件特殊要求

| 检查项 | 要求 | 关联标准 |
|--------|------|----------|
| MISRA C++合规 | 100%合规 | ISO 26262 |
| 空指针检查 | 0容忍 | 运行时安全 |
| 资源泄漏检查 | 0容忍 | 系统稳定性 |
| 线程安全 | 强制检查 | 并发安全 |
| 硬编码密钥 | 禁止出现 | 信息安全 |

## 3. 工具配置详情

### 3.1 SonarQube 配置

**配置文件**: `sonar-project.properties`

#### 基本项目信息
- 项目标识: `cockpit-main-interaction`
- 显示名称: `智能座舱主交互系统`
- 源码路径: `app/src/main/java`, `domain/src/main/java`
- 测试代码路径: `*/src/test/*`, `*/src/androidTest/*`

#### 语言与编码
- 主语言: Kotlin
- 文件编码: UTF-8
- 源码版本: Java 17

#### 代码覆盖率
- 最小行覆盖率: 80%
- 最小分支覆盖率: 70%
- 单元测试覆盖率目标: 80%

#### 排除规则
```
# 自动生成的代码
**/generated/**
**/build/**
**/*Test*/**

# 第三方库
**/libs/**
```

### 3.2 Detekt 配置

**配置文件**: `detekt-config.yml`

#### 复杂度规则
| 规则 | 阈值 | 说明 |
|------|------|------|
| LongMethod | 60行 | 函数最大行数 |
| LongParameterList | 6个 | 参数最大数量 |
| LargeClass | 150行 | 类最大行数 |
| TooManyFunctions | 11个 | 类最大函数数 |
| CyclomaticComplexMethod | 15 | 圈复杂度 |
| NestedBlockDepth | 4层 | 嵌套深度 |

#### 代码风格规则
| 规则 | 要求 |
|------|------|
| NamingConvention | 严格执行命名规范 |
| MagicNumber | 禁止魔法数字 |
| WildcardImport | 禁止通配符导入 |
| UnusedImports | 禁止未使用导入 |
| MaxLineLength | 120字符 |

#### 潜在缺陷规则
| 规则 | 级别 |
|------|------|
| LateinitUsage | 警告 |
| UnreachableCode | 错误 |
| UnsafeCallOnNullableType | 阻塞级 |
| RedundantSuspendModifier | 警告 |
| SuspendFunWithCoroutineScopeReceiver | 警告 |

### 3.3 Android Lint 配置

#### 严重级别配置
```xml
<lint>
    <!-- 错误级别 (阻塞) -->
    <issue id="NewApi" severity="error" />
    <issue id="InlinedApi" severity="error" />
    <issue id="UnusedResources" severity="error" />
    
    <!-- 警告级别 -->
    <issue id="ObsoleteSdkInt" severity="warning" />
    <issue id="Overdraw" severity="warning" />
    <issue id="UnusedAttribute" severity="warning" />
    
    <!-- 禁用 (非项目相关) -->
    <issue id="GoogleAppIndexingWarning" severity="ignore" />
    <issue id="UnusedIds" severity="ignore" />
</lint>
```

#### 关键检查项
- **API兼容性**: 检查API级别调用
- **性能优化**: 过度绘制、内存泄漏
- **资源管理**: 未使用资源、重复资源
- **安全性**: 硬编码密码、不安全的网络配置

### 3.4 ktlint 配置

#### 代码风格规范
| 规则 | 配置 |
|------|------|
| 缩进 | 4个空格 |
| 换行符 | LF |
| 最大行长度 | 120字符 |
| 导入排序 | ASCII顺序 |
| 尾部空格 | 禁止 |

#### 启用规则
- `standard:indent` - 缩进规范
- `standard:max-line-length` - 行长度限制
- `standard:no-wildcard-imports` - 禁止通配符导入
- `standard:final-newline` - 文件末尾空行
- `standard:import-ordering` - 导入排序

## 4. CI/CD 集成方案

### 4.1 流水线阶段

```yaml
stages:
  - lint          # ktlint + Detekt
  - compile       # Android Lint (编译时)
  - test          # 单元测试
  - sonar         # SonarQube分析
  - quality-gate  # 质量门禁检查
```

### 4.2 质量门禁执行流程

```
┌─────────────────────────────────────────────────────────────┐
│  Developer Commit                                           │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Pre-commit Hook                                            │
│  ├── ktlint check (代码风格)                                │
│  └── Detekt quick check (复杂度)                            │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  CI Pipeline                                                │
│  ├── Detekt Full Analysis                                   │
│  ├── Android Lint                                           │
│  ├── Unit Test (覆盖率80%)                                   │
│  └── SonarQube Scan                                         │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────┐
│  Quality Gate Check                                         │
│  ├── 阻塞问题 = 0 ?                                         │
│  ├── 严重问题 ≤ 5 ?                                         │
│  ├── 主要问题 ≤ 50 ?                                        │
│  ├── 重复率 ≤ 3% ?                                          │
│  └── 技术债务 ≤ 5天 ?                                       │
└───────────────────────────┬─────────────────────────────────┘
                            ▼
              ┌─────────────┴─────────────┐
              ▼                           ▼
    ┌─────────────────┐         ┌─────────────────┐
    │   PASS → 合并   │         │   FAIL → 阻断   │
    └─────────────────┘         └─────────────────┘
```
