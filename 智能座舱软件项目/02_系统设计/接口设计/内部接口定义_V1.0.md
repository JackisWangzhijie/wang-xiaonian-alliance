本文档定义智能座舱主交互系统各模块间的内部接口规范，采用Java/Kotlin语言进行接口定义，确保模块间通信的标准化和可追溯性。

## 2. 接口设计原则

### 2.1 设计准则
1. **单一职责原则**: 每个接口只负责一类功能
2. **开闭原则**: 接口对扩展开放，对修改关闭
3. **依赖倒置**: 依赖抽象接口，而非具体实现
4. **异步优先**: 耗时操作采用异步回调或协程
5. **空安全**: Kotlin接口必须处理空安全

### 2.2 命名规范
- 接口名: `I` + 模块名 + `Service`/`Manager`/`Provider`
- 方法名: 动词 + 名词，驼峰命名
- 常量名: 全大写，下划线分隔
- 回调接口: `On` + 动作 + `Listener`/`Callback`

### 2.3 安全等级标识
| 标识 | ASIL等级 | 说明 |
|------|----------|------|
| `@ASIL_B` | ASIL B | 高安全等级接口 |
| `@ASIL_A` | ASIL A | 中安全等级接口 |
| `@QM` | QM | 质量管理等级 |

     * 获取剩余油量百分比 (0-100)

/**

interface IDrivingRestrictionService {

     * 检查当前是否处于行驶状态
     */

     * 检查应用是否被允许运行

     * 获取当前限制级别

     * 请求临时解除限制（仅用于紧急场景）
     * @param reason 解除原因
     * @param durationMs 解除时长（毫秒）
     * @param callback 结果回调
     */
    fun requestTemporaryExemption(
        reason: String,
        durationMs: Long,
        callback: ExemptionCallback
    )
    
    /**
     * 添加限制状态监听器
     */
    fun addRestrictionListener(listener: RestrictionListener)
    
    /**
     * 移除限制状态监听器
     */
    fun removeRestrictionListener(listener: RestrictionListener)
    
    /**
     * 获取白名单应用列表
     */
    fun getWhitelistApps(): List<String>
    
    /**
     * 更新白名单（需系统权限）
     */
    fun updateWhitelist(apps: List<String>): Boolean
}

 * 限制结果
 */
data class RestrictionResult(
    val allowed: Boolean,
    val action: RestrictionAction,
    val reason: String
)

/**
 * 限制动作枚举
 */
enum class RestrictionAction {
    ALLOW,           // 允许运行
    PAUSE,           // 暂停运行
    RESTRICT_UI,     // 限制交互
    FORCE_HOME,      // 强制返回桌面
    GRAY_DISPLAY,    // 灰度显示
    BLOCK            // 完全禁止
}

/**
 * 限制级别
 */
enum class RestrictionLevel(val value: Int) {
    NONE(0),         // 无限制
    LOW(1),          // 低限制
    MEDIUM(2),       // 中限制
    HIGH(3),         // 高限制
    EMERGENCY(4)     // 紧急限制
}

/**
 * 限制状态监听器
 */

    fun onRestrictionChanged(level: RestrictionLevel)
    fun onAppRestricted(packageName: String, action: RestrictionAction)
}
