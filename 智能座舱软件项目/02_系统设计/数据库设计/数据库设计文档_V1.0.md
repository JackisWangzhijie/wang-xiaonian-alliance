## 1. 引言

### 1.1 目的
本文档定义了智能座舱主交互系统的数据库结构，包括表结构、索引、约束和数据字典，用于指导系统开发和数据持久化实现。

### 1.2 范围
数据库设计覆盖以下核心功能模块：
- 消息中心模块（消息存储、历史记录、优先级管理）
- 应用管理模块（应用信息、白名单、应用状态）
- 用户设置模块（用户偏好、个性化设置）
- 导航模块（导航历史、收藏夹、离线数据）
- 系统配置模块（系统参数、功能开关、OTA配置）

### 1.3 设计原则
1. **数据完整性**: 使用主键、外键、约束确保数据完整性
2. **性能优先**: 针对车载环境优化，查询响应时间≤100ms
3. **安全性**: 敏感数据加密存储，符合ISO 26262要求
4. **可扩展性**: 预留字段支持未来功能扩展
5. **可追溯性**: 所有表包含创建/更新时间戳

## 2. 数据库架构设计

### 2.1 数据库选型
| 属性 | 规格 |
|------|------|
| 数据库类型 | SQLite 3.36+ |
| 存储位置 | /data/data/com.longcheer.cockpit/databases/ |
| 默认编码 | UTF-8 |
| 事务支持 | 完全支持ACID |
| 最大容量 | 2TB（理论）/ 实际限制由存储决定 |
| 并发控制 | 读写分离，WAL模式 |

### 2.2 数据库文件划分
| 数据库文件 | 用途 | 预估大小 |
|-----------|------|----------|
| cockpit_main.db | 核心业务数据 | 100-500MB |
| cockpit_nav.db | 导航数据（含离线地图索引） | 1-5GB |
| cockpit_cache.db | 临时缓存数据 | 50-200MB |

### 2.3 表空间规划
```
cockpit_main.db
├── 消息相关表 (message_*) - 约20%存储
├── 应用相关表 (app_*) - 约15%存储
├── 用户相关表 (user_*) - 约10%存储
├── 系统配置表 (sys_*) - 约5%存储
└── 日志审计表 (log_*) - 约50%存储（可清理）
```

### 3.2 核心实体说明

#### 3.2.1 消息实体
- **主实体**: message（消息表）
- **关联实体**: 
  - message_category（消息分类表）- 1:N
  - message_attachment（消息附件表）- 1:N
  - sys_user（用户表，接收者）- N:1

#### 3.2.2 应用实体
- **主实体**: application（应用信息表）
- **关联实体**:
  - app_category（应用分类表）- N:1
  - app_whitelist（应用白名单表）- 1:1
  - app_status（应用状态表）- 1:1

#### 3.2.3 用户实体
- **主实体**: sys_user（用户表）
- **关联实体**:
  - user_settings（用户设置表）- 1:N
  - user_profile（用户档案表）- 1:1

#### 3.2.4 导航实体
- **主实体**: nav_history（导航历史表）
- **关联实体**:
  - nav_favorite（收藏地点表）- 1:N
  - nav_route（路线记录表）- 1:N

## 5. 索引设计

### 5.1 索引策略
1. **主键索引**: 所有表自动创建主键索引
2. **唯一索引**: 业务唯一键创建唯一索引
3. **查询索引**: 高频查询字段创建普通索引
4. **复合索引**: 多条件查询创建复合索引
5. **覆盖索引**: 常用查询字段组合创建覆盖索引

### 5.2 索引明细

#### 5.2.1 消息表索引
```sql
-- 业务消息ID唯一索引
CREATE UNIQUE INDEX idx_msg_msg_id ON message(msg_id);

-- 用户消息查询索引(用户ID+是否删除+创建时间倒序)
CREATE INDEX idx_msg_user_query ON message(user_id, is_deleted, create_time DESC);

-- 消息分类查询索引
CREATE INDEX idx_msg_category ON message(category_id, is_deleted);

-- 优先级查询索引
CREATE INDEX idx_msg_priority ON message(priority, is_read);

-- 未读消息查询索引
CREATE INDEX idx_msg_unread ON message(user_id, is_read, is_deleted) WHERE is_read = 0;

-- 过期消息清理索引
CREATE INDEX idx_msg_expire ON message(expire_time) WHERE expire_time IS NOT NULL;

-- 来源应用查询索引
CREATE INDEX idx_msg_source ON message(source_app, create_time DESC);
```

#### 5.2.2 应用信息表索引
```sql
-- 应用包名唯一索引
CREATE UNIQUE INDEX idx_app_app_id ON application(app_id);

-- 分类查询索引
CREATE INDEX idx_app_category ON application(category_id, is_enabled);

-- 启动次数排序索引
CREATE INDEX idx_app_launch ON application(launch_count DESC);

-- 白名单查询索引
CREATE INDEX idx_app_whitelist ON application(is_in_whitelist, is_enabled);

-- 更新查询索引
CREATE INDEX idx_app_update ON application(last_update DESC);
```

#### 5.2.3 用户设置表索引
```sql
-- 用户设置唯一索引
CREATE UNIQUE INDEX idx_setting_user_key ON user_settings(user_id, setting_key);

-- 分类查询索引
CREATE INDEX idx_setting_category ON user_settings(user_id, category);

-- 同步状态索引
CREATE INDEX idx_setting_sync ON user_settings(is_synced) WHERE is_synced = 0;
```

#### 5.2.4 导航历史表索引
```sql
-- 用户历史查询索引
CREATE INDEX idx_nav_user ON nav_history(user_id, is_deleted, start_time DESC);

-- 收藏查询索引
CREATE INDEX idx_nav_favorite ON nav_history(user_id, is_favorite) WHERE is_favorite = 1;

-- 终点POI索引
CREATE INDEX idx_nav_end_poi ON nav_history(end_poi, user_id);

-- 时间范围查询索引
CREATE INDEX idx_nav_time ON nav_history(start_time);
```

#### 5.2.5 收藏地点表索引
```sql
-- 用户收藏查询索引
CREATE INDEX idx_fav_user ON nav_favorite(user_id, fav_type, sort_order);

-- 坐标查询索引
CREATE INDEX idx_fav_location ON nav_favorite(lat, lng);

-- 同步状态索引
CREATE INDEX idx_fav_sync ON nav_favorite(is_synced) WHERE is_synced = 0;
```

#### 5.2.6 系统配置表索引
```sql
-- 配置键唯一索引
CREATE UNIQUE INDEX idx_config_key ON system_config(config_key);

-- 分类查询索引
CREATE INDEX idx_config_category ON system_config(category);
```

#### 5.2.7 日志表索引
```sql
-- 日志时间索引
CREATE INDEX idx_log_time ON operation_log(log_time DESC);

-- 用户操作索引
CREATE INDEX idx_log_user ON operation_log(user_id, log_time DESC);

-- 模块操作索引
CREATE INDEX idx_log_module ON operation_log(module, action, log_time DESC);

-- 事件查询索引
CREATE INDEX idx_event_query ON system_event(event_time DESC, event_level);
```

## 7. 安全设计

### 7.1 数据加密策略
| 数据类型 | 加密方式 | 密钥管理 |
|----------|----------|----------|
| 用户手机号 | AES-256-GCM | 硬件安全模块(HSM) |
| 用户邮箱 | AES-256-GCM | 硬件安全模块(HSM) |
| 应用签名 | SHA-256哈希 | 系统内置公钥 |
| 敏感配置 | AES-256-GCM | 系统密钥库 |

### 7.2 访问控制
1. **数据库文件权限**: 仅限系统用户访问(rw-------)
2. **敏感表访问**: 通过ContentProvider控制访问权限
3. **SQL注入防护**: 所有查询使用参数化SQL
4. **日志脱敏**: 日志中不记录敏感信息

### 7.3 数据完整性
1. **外键约束**: 启用SQLite外键检查(PRAGMA foreign_keys = ON)
2. **事务保护**: 关键操作使用事务保证ACID
3. **数据校验**: 应用层对关键数据进行校验

### 7.4 隐私保护
1. **数据最小化**: 仅收集必要数据
2. **数据匿名化**: 日志使用匿名用户ID
3. **数据保留期**: 消息30天自动清理，日志7天自动清理
4. **用户授权**: 敏感操作需用户确认

## 8. 性能优化策略

### 8.1 查询优化
1. **覆盖索引**: 高频查询使用覆盖索引减少回表
2. **查询缓存**: 常用配置数据内存缓存
3. **分页查询**: 大数据集使用分页，避免全表扫描
4. **预编译语句**: 重复查询使用预编译SQL

### 8.2 写入优化
1. **批量插入**: 多条记录批量插入
2. **延迟写入**: 非关键数据延迟批量写入
3. **WAL模式**: 启用Write-Ahead Logging提升并发
4. **事务合并**: 小事务合并为大事务

### 8.3 存储优化
1. **数据压缩**: 大文本字段使用压缩存储
2. **自动清理**: 过期数据自动清理
3. **VACUUM**: 定期执行VACUUM释放空间
4. **分区存储**: 大表按时间分区

### 8.4 性能指标
| 操作类型 | 目标响应时间 | 最大响应时间 |
|----------|-------------|-------------|
| 消息查询 | ≤50ms | ≤100ms |
| 应用查询 | ≤30ms | ≤50ms |
| 设置读取 | ≤10ms | ≤20ms |
| 导航历史查询 | ≤100ms | ≤200ms |
| 批量插入(100条) | ≤500ms | ≤1000ms |

## 9. 备份与恢复策略

### 9.1 备份策略
| 备份类型 | 频率 | 保留周期 | 存储位置 |
|----------|------|----------|----------|
| 完整备份 | 每周 | 4周 | 云端存储 |
| 增量备份 | 每天 | 7天 | 本地存储 |
| 关键配置 | 实时 | 永久 | 云端+本地 |

### 9.2 恢复策略
1. **自动恢复**: 启动时检测数据库完整性，损坏时自动恢复
2. **云端同步**: 用户设置自动同步云端，换机可恢复
3. **版本兼容**: 支持数据库版本升级和数据迁移

### 9.3 灾难恢复
1. **数据库损坏**: 从最近备份恢复，丢失数据≤1天
2. **存储故障**: 切换到备用存储，服务不中断
3. **数据误删**: 支持30天内数据恢复
