| 分支 | 状态 | 用途 |
|------|------|------|
| `main` | ✅ **主分支** | **所有人统一在此分支提交** |

## 🔄 操作流程

```bash
# 1. 每次开始前，先拉取最新main
 git pull origin main

# 2. 修改文件...

# 3. 提交并推送
 git add .
 git commit -m "你的提交信息"
 git push origin main
```

## ⚠️ 注意事项

1. **不要**创建个人分支（如 `member-claw-updates`）
2. **不要**在 `master` 分支提交
3. 如有冲突，先 `pull` 再解决，然后 `push`
